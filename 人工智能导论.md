# 《人工智能导论》课程项目说明

## 0. 统一规则（所有项目通用）

### 0.1 允许的工具与语言

* **语言**：Python 3.x
* **允许库**（按需使用）：

  * 通用：numpy、pandas、scikit-learn、matplotlib、scipy、tqdm
  * 深度学习：torch、torchvision（如需）
  * Boosting：xgboost / lightgbm（如需）
  * 可解释：shap（如需）
  * 强化学习：gymnasium（RL 项目必须）([gymnasium.farama.org][1])
  * NLP/Agent（如需）：transformers、sentence-transformers、faiss-cpu、chromadb
* **禁止**：使用任何“直接生成完整答案/完整代码/完整报告正文”的自动化生成内容（包括但不限于整套训练脚本、整段报告）。

  * **允许**：查资料、解释概念、定位报错、局部代码建议。
  * **必须**：在报告“诚信声明”中写明你使用了哪些辅助工具与使用范围（例如：仅用于解释概念、仅用于 Debug）。

### 0.2 数据与可复现性（必须）

* 必须提供 `requirements.txt` 或 `environment.yml`
* 必须固定随机种子（至少：numpy、torch/sklearn；RL 还必须固定环境 seed）
* 必须写清楚数据来源（链接）与下载方式（脚本或步骤）
* **必须避免数据泄漏**（如：同一对象/病例/同源样本不得跨 train/test；时间序列禁止随机打乱）

### 0.3 交付物（必须全部提交）

1. **代码**：可一键运行（`python main.py`）
2. **报告**：`report.pdf`
3. **结果文件**（二选一，按项目类型提交）：

   * 分类/回归：`predictions.csv`（必须含 `id` 与 `y_pred`；分类可额外含 `prob`）
   * 强化学习：`learning_curve.csv`（必须含 `episode, return`）
4. **复现实验脚本**：`run.sh` 或 `run.bat`（二选一）

### 0.4 报告固定结构（必须按此顺序写）

1. 问题定义（任务、输入输出、评价指标）
2. 数据（来源、规模、预处理、划分方式）
3. 方法（基线模型 + 改进模型，说明原理与实现细节）
4. 实验设置（超参、训练细节、硬件/时间可选）
5. 结果（表格 + 图；必须含对比与消融）
6. 讨论（错误分析/失败案例 ≥ 3 个）
7. 结论（不超过 10 行）
8. 诚信声明（是否使用辅助工具、使用范围）

### 0.5 评分标准（100 分）

* 完整性与可复现（20）
* 基线正确 + 改进有效（25）
* 评估严谨（数据划分正确、指标正确）（20）
* 分析与讨论（消融、失败案例、解释）（25）
* 工程质量（结构清晰、代码可读、输出规范）（10）

## 1) 机器学习方向（Machine Learning）— 4 个项目

* 长期记忆：向量库（FAISS/Chroma/自写）+ 检索注入
* 对比三种设置（同一测试集）：

  1. 无记忆
  2. 仅短期
  3. 短期+长期
* 指标（必须两项）：

  * 多轮一致性（给出可执行判定 + 标注表）
  * 信息遗忘率（给出可执行判定 + 标注表）

**拓展方向（加分）**

* 记忆压缩（summary memory/聚类）
* 错误记忆污染：≥10 案例（错误写入导致后续错误）

**额外验收物**

* `memory_store/`（向量库或导出文件）
* `conversation_tests.jsonl`（≥30 组多轮脚本）

### PJ-AG3 多 Agent 协作任务系统

**目标**：两个或多个 Agent 分工协作完成复杂任务（如报告生成/调研总结）。

**基础要求（必须全部完成）**

* 至少 2 个 Agent（推荐 3 个：Planner/Researcher/Writer）
* 角色 Prompt 清晰且固定
* 通信协议：结构化消息格式（JSON schema 或固定字段）
* 日志：完整记录每轮交互
* 任务集：≥10 个复杂任务（每个任务至少 3 个子步骤）

**拓展方向（加分）**

* 冲突意见注入并分析系统处理方式
* 投票/仲裁机制（Judge agent / majority vote）

**额外验收物**

* `protocol.md`（消息格式定义）
* `multi_agent_logs/`（≥10 个任务完整日志）

### PJ-AG4 LLM 驱动的博弈/市场模拟 Agent

**目标**：多个 LLM Agent 参与重复博弈/市场交易模拟并统计结果。

**基础要求（必须全部完成）**

* 至少 **3 个独立 Agent**
* 定义收益函数与规则（必须可代码复现）
* 交互轮数：不少于 **10 轮**
* 输出统计：每个 agent 的累计收益；并至少 1 张可视化（曲线/柱状图等）

**拓展方向（加分）**

* 声誉机制（信誉影响合作/交易）
* 有限理性限制（限制上下文长度/信息/加入噪声）

**额外验收物**

* `simulation_results.csv`（round, agent_action, payoff 等）
* `strategy_analysis.pdf`（或写入报告内专章）

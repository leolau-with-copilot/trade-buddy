# Repository Guidelines

## Project Structure & Module Organization
- `meeting_agents.py` contains the three-stage AutoGen workflow and streaming discussion logic.
- `prompts.py` defines agent roles, selector rules, and moderator resolution-card format.
- `webapp.py` serves the FastAPI API and static frontend.
- `frontend/` contains the visual dashboard: `index.html`, `styles.css`, and `app.js`.
- `sample_input.json` provides a ready-to-run activity-planning example.
- `ITERATIONS.md` and `SESSION_REPORT_2026-04-25.md` track progress and handoff context.

## Build, Test, and Development Commands
Run commands from the repository root.

- `python -m venv .venv && source .venv/bin/activate` — create and activate a local virtual environment.
- `pip install -r requirements.txt` — install AutoGen, FastAPI, and model client dependencies.
- `export DEEPSEEK_API_KEY=your_key` — set the model API key before running the app.
- `uvicorn webapp:app --reload` — start the visual web app at `http://127.0.0.1:8000`.
- `python main.py --input sample_input.json` — run the planner from the command line and save outputs to `outputs/`.
- `python -m py_compile main.py meeting_agents.py webapp.py prompts.py` — quick syntax check for the core app.

## Coding Style & Naming Conventions
Use Python 3 style conventions: 4-space indentation, `snake_case` for functions/files, `PascalCase` for classes, and concise docstrings only where behavior is non-obvious. No formatter or linter config is committed yet, so follow PEP 8 and keep imports grouped as standard library, third-party, then local. Name Alembic revisions with short, descriptive messages such as `add_user_table`.

## Testing Guidelines
There is no committed test suite yet. For new logic, add tests under `tests/` using `pytest`, with filenames like `test_<feature>.py`. For schema changes, always verify `alembic -c my-app/alembic.ini upgrade head` succeeds on a fresh database and review autogenerated migrations before committing.

## Commit & Pull Request Guidelines
Git history is minimal (`sub`), so prefer clear imperative commit subjects going forward, e.g. `Add initial user schema`. Keep commits focused and small. PRs should include: a short summary, affected paths, migration notes, manual verification steps, and screenshots only if a UI is added later.

## Configuration & Data Hygiene
Do not commit real API keys. Keep `DEEPSEEK_API_KEY` in your shell environment only. Do not edit `__pycache__/` or commit generated output files unless they are intentional demo artifacts.

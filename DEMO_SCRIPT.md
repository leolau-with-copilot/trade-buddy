# 演示讲稿（Demo Script）

## 1. 开场介绍
大家好，我们做的是一个 **基于 AutoGen 的多 Agent 活动策划系统**。  
它的目标不是让单个模型直接给出一份活动方案，而是模拟一个真实的策划团队，通过多个不同角色的 Agent 先讨论，再生成结果。

---

## 2. 展示目标
今天主要展示 3 个能力：
1. 系统如何根据输入自动生成筹备会议方案
2. 多 Agent 如何像开会一样动态讨论
3. 系统如何输出最终活动策划书，并在前端实时可视化

---

## 3. 系统流程介绍
这个系统分成三个阶段：

### 第一阶段：筹备会议
系统会先根据活动需求，生成一份内部筹备会议方案，包括：
- 要讨论什么
- 谁应该参加
- 哪些问题必须先敲定

### 第二阶段：多 Agent 讨论
接着多个 Agent 开始模拟讨论：
- 总策划关注创意和亮点
- 执行统筹关注落地和流程
- 宣传传播关注吸引力和转化
- 预算风险关注资源和风险
- moderator 负责收束观点并输出决议卡

### 第三阶段：策划书输出
最后系统会综合讨论结果，生成一份可执行的活动策划书。

---

## 4. 前端展示说明
在前端界面中，我们可以看到：
- 左侧是活动输入区
- 中间上方是虚拟 Agent 舱室
- 实时发言时会高亮当前说话角色
- 下方会滚动显示讨论内容
- moderator 的阶段性结论会形成“本轮决议”卡片
- 最终会输出完整活动策划书

---

## 5. 技术亮点
这个项目的几个亮点在于：

### 亮点 1：不是单 Agent，而是多 Agent
系统通过角色分工，模拟一个更接近真实策划团队的工作方式。

### 亮点 2：不是固定轮流，而是动态选择发言者
我们使用了 `SelectorGroupChat`，它会根据当前对话内容动态决定谁最适合继续发言。

### 亮点 3：可视化展示推理过程
相比只输出结果，这个系统还能展示中间讨论过程，让用户更容易理解方案是怎么形成的。

---

## 6. 当前成果
目前系统已经实现：
- AutoGen 多 Agent 编排
- DeepSeek 模型接入
- 三阶段流程
- 流式消息显示
- 虚拟角色发言面板
- moderator 决议卡
- Markdown 策划书展示
- 导出与历史任务回放

---

## 8. 结束语
总体来说，这个项目验证了一种新的思路：

> 通过多 Agent 协作 + 可视化前端，让 AI 不只是“生成结果”，而是参与一个更像真实团队协作的策划过程。
